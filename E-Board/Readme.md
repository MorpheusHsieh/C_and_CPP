一、實驗目的：學習組合語言和硬體 I/O之間理論和實際的結合。

二、實驗方式：本實驗以組合語言為主Ｃ語言為輔。

三、實驗原理：電子看版以8255之PORT A, B控制列的輸出，以Port
              C 控制行的輸出。
              Port A的位址為 03E0H、Port B的位址為 03E1H、
              Port C的位址為 03E2H。
              8255位址為 03E3H。送出080H時設定為輸出。

四、程式說明：
              Ｃ語言部份說明如流程圖。
              PAINT.EXE    - PAINT 主程式
              PAINT.CPP    - PAINT 原始碼
              PAINT.DOC    - PAINT 說明書

              組合語言部份包含上、下、左、右四種移動。
              LEDUP.DAT    - 上移原始程式
              LEDOWN.DAT   - 下移原始程式
              LEDLEFT.DAT  - 左移原始程式
              LEDRIGHT.DAT - 右移原始程式
              MASM.EXE     - 組合語言 Compiler V5.0
              LINK.EXE     - 組合語言 Link     V5.0

              批次檔部份
              INSTALL.BAT  - 載入程式

五、實驗心得：
              一、MOUSE 控制部份：如果以單 KEY來控制電子看版
                  LED 的 ON/OFF ，當 MOSUE KEY按下同時取得游
                  標點的顏色值，將會由於迴圈過快而產生取得零
                  值的誤動作，所以應以 MOUSE KEY放開時取游標
                  點的顏色值便可以解決。
              二、因為要使程式簡單，所以採用 BGI及FONT內含的
                  方式。
              三、為配合不同解析度的螢幕所以採用自動偵測及強
                  制定義的方式。
              四、電子看板及MENU選項採用範圍或游標點顏色取樣
                  判斷。
              五、以 I/O方式將會造成開檔失敗所以全改為標準模
                  式開檔。
              六、為防止 NULL POINTER 所以陣列開啟比實際使用
                  值大。
              七、倚天字型庫資料排列為橫列式，而電子看版為直
                  列式，所以需要轉碼。
              八、將電子看版程式看為主程式相同，而展示程式不
　　　　　　　　　同，所以將展示程式視為一個個函數庫，需要時
                  才連結起來，但因時間不足只能做簡單的範例。
              九、為了連結的方便所以放棄了TIMY模式，而將程式
              　　、資料、堆疊段分開及重寫。　

十、實驗結論：
                  認為重要的是電子看版的控制，所以發展方向應
              放於不求程式的長短而應著重於不同展示模式的設計
              ，以增加函數館的方式，假以時日可發展成套裝軟體
              ，屆時電子看版的實用性將可大幅度的提高。
